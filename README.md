# Katelyn Fajardo — Portfolio

A static portfolio site. No build step, no dependencies — just HTML, CSS, and images.

## Files
- `index.html` — the page
- `style.css` — all styling (Poppins + the blue palette: #2C5EAD #1591DC #4BB8FA #C4E2F5)
- `assets/` — all project images, optimized for web

## Deploy on GitHub Pages
1. Create a new repository (e.g. `portfolio`).
2. Upload these files, keeping the folder structure (`index.html` and `style.css` at the root, images inside `assets/`).
3. Repo **Settings → Pages → Build and deployment → Source: Deploy from a branch**.
4. Pick branch `main` and folder `/ (root)`, then **Save**.
5. Your site goes live at `https://<your-username>.github.io/portfolio/` within a minute or two.

> Tip: for a clean URL like `https://<your-username>.github.io/`, name the repo `<your-username>.github.io` instead.

## Editing
- Text lives in `index.html` — search for the project name and edit the paragraph.
- Colors/spacing/fonts live at the top of `style.css` under `:root`.
- To swap an image, drop a new file into `assets/` and update the matching `src="assets/..."` in `index.html`.
